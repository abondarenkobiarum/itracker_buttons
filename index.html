Style guide

Version 1.0

Buttons

Buttons initiate specific actions and have predefined styles and sizes.

Navigation buttons

Navigation buttons should only be used when appearing independent of a toolbar, or ‘floating’ on the page.

Push buttons

Special buttons should only be used to reinforce positive behavior, or as call-to-action (CTA) buttons guiding users towards a goal conversion. Danger buttons should only be used for severe or negative consequences, such as “delete”. They should always be accompanied by an alert popup before proceeding. Link buttons should only be used on pop ups and pagination. They should always be right or centre aligned. All other variants should be used in the correct hierarchy and style.

Always

  • Use verbs in titles
  • Use sentence case for titles
  • Keep titles short and specific
  • Use simple present tense, for example "Buy now" or "Use test"
  • Use Semi Bold weight, 20px kerning

Size and application

Push buttons may include a title or an icon. They should never contain an image. They have multiple states with corresponding size, color and type specifications as indicated adjacent. When a button appears as part of a group (2 or more buttons), the button widths should be equal in size. In all other cases, specified width, height and gutter requirements must apply.
